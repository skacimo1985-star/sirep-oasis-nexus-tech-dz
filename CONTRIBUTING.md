## How to Contribute

### Reporting Bugs

1. Check if the issue already exists in the [Issues](../../issues) section
2. Open a new issue with a clear title and description
3. Include steps to reproduce, expected vs actual behavior
4. Add relevant logs or screenshots

### Requesting Features

1. Open an issue with the `enhancement` label
2. Describe the feature and its business value
3. Await approval from the project maintainer before implementing

### Submitting Code

Only authorized contributors may submit pull requests:

1. Fork the repository (authorized contributors only)
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Follow existing code style (TypeScript strict mode, ESLint rules)
4. Write or update tests as needed
5. Ensure CI pipeline passes before submitting PR
6. Submit a Pull Request with a detailed description

## Code Standards

- **Language**: TypeScript (strict mode)
- **Frontend**: React + Vite + TailwindCSS
- **Backend**: Node.js + Express
- **Linting**: ESLint (no warnings allowed in CI)
- **Commits**: Use conventional commit format (`feat:`, `fix:`, `docs:`, `chore:`)

## Branching Strategy

- `main` - production-ready code only
- `develop` - integration branch
- `feature/*` - new features
- `fix/*` - bug fixes
- `hotfix/*` - urgent production fixes
